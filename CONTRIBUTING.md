# Guia de Contribuição 🚀

Oba! Que bom que você quer contribuir com a Caramelo Tech! 💛

Toda contribuição é bem-vinda, não importa o tamanho. Crescemos juntos!

## 🍮 Como Contribuir

### ⭐ Dê uma estrela

A forma mais simples de contribuir! Dê uma estrela nos repositórios que você curtir.

### 🐛 Encontrou um bug?

1. Verifique se já existe uma [issue](../../issues) sobre o problema
2. Se não existir, abra uma nova issue usando o template de **Bug Report**
3. Descreva o problema com o máximo de detalhes possível

### 💡 Tem uma ideia?

1. Confira as [issues](../../issues) para ver se alguém já sugeriu algo parecido
2. Abra uma nova issue usando o template de **Feature Request**
3. Descreva sua ideia - adoramos novas sugestões!

### 🔧 Quer codar?

1. Faça um fork do repositório
2. Crie uma branch para sua feature/fix:
   ```bash
   git checkout -b feature/minha-feature
   # ou
   git checkout -b fix/meu-fix
   ```
3. Faça suas alterações
4. Commit suas mudanças seguindo o padrão [Conventional Commits](https://www.conventionalcommits.org/):
   ```bash
   git commit -m "feat: adiciona nova funcionalidade"
   git commit -m "fix: corrige bug no componente X"
   git commit -m "docs: atualiza documentação"
   ```
5. Push para sua branch:
   ```bash
   git push origin feature/minha-feature
   ```
6. Abra um Pull Request 🎉

## 📝 Padrões do Projeto

### Commits

Usamos [Conventional Commits](https://www.conventionalcommits.org/):

| Tipo       | Descrição                        |
| ---------- | -------------------------------- |
| `feat:`    | Nova funcionalidade              |
| `fix:`     | Correção de bug                  |
| `docs:`    | Documentação                     |
| `refactor:`| Refatoração de código            |
| `test:`    | Adição ou correção de testes     |
| `chore:`   | Tarefas de manutenção            |

### Branches

- `feature/` - Nova funcionalidade
- `fix/` - Correção de bug
- `docs/` - Documentação
- `refactor/` - Refatoração

## 🤝 Código de Conduta
